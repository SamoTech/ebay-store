## UX Principles

### Clarity Over Cleverness
- Users should instantly understand what to do
- Obvious > Clever (don't make users think)
- Use familiar patterns (don't reinvent the wheel)
- Clear labels, no jargon or ambiguous terms

### Minimize Friction
- Reduce steps to complete tasks
- Autofill, defaults, and smart suggestions
- Clear error messages with recovery paths
- Progress indicators for multi-step flows

### Design for Speed
- Perceived speed > actual speed (use skeletons, optimistic UI)
- Lazy load content below the fold
- Show instant feedback on user actions
- Optimize images and assets

### Mobile-First Mindset
- Design for smallest screen first, scale up
- Touch targets ≥ 44×44px (Apple) or 48×48px (Google)
- Responsive layouts, not separate mobile sites
- Test on real devices, not just emulators

## Design Workflow

### Phase 1: Research & Discovery
1. Understand user needs and pain points
2. Analyze competitor experiences
3. Review analytics and user feedback
4. Define success metrics (task completion rate, time on task, etc.)

### Phase 2: Information Architecture
1. Map user flows and journeys
2. Define page hierarchy and navigation
3. Create sitemaps and wireframes
4. Identify key conversion paths

### Phase 3: Interface Design
1. Create high-fidelity mockups
2. Design for all states (empty, loading, error, success)
3. Ensure visual hierarchy and readability
4. Use consistent spacing, typography, and colors

### Phase 4: Prototyping & Testing
1. Build interactive prototypes
2. Conduct usability testing with real users
3. Iterate based on feedback
4. Validate with data (A/B tests, analytics)

## UX Design Checklist

For every screen or feature:

- [ ] **Clear Purpose**: User knows what this page/feature does
- [ ] **Obvious CTA**: Primary action is visually prominent
- [ ] **Readable Text**: Font size ≥ 16px, high contrast (WCAG AA)
- [ ] **Touch-Friendly**: Buttons/links ≥ 44×44px
- [ ] **Responsive**: Works on mobile, tablet, desktop
- [ ] **Fast Loading**: Perceived speed optimized (skeletons, lazy load)
- [ ] **Error Handling**: Clear error messages with next steps
- [ ] **Accessible**: Keyboard navigable, screen reader friendly
- [ ] **Consistent**: Follows established design patterns
- [ ] **Tested**: Validated with real users or data

## Conversion Optimization Tactics

### Forms
- Minimize fields (only ask for essentials)
- Use smart defaults and autofill
- Inline validation (show errors immediately)
- Clear labels and placeholder text
- Show progress in multi-step forms

### CTAs (Call to Action)
- Use action-oriented text ("Get Started" > "Submit")
- High contrast colors (stands out from page)
- Above the fold on key pages
- Repeat CTAs for long pages

### Trust Signals
- Show testimonials, reviews, ratings
- Display security badges (SSL, payment logos)
- Transparent pricing (no hidden fees)
- Clear return/refund policies

### Urgency & Scarcity
- Limited-time offers ("Sale ends tonight")
- Stock indicators ("Only 3 left")
- Social proof ("125 people viewing this")

## Accessibility Guidelines

### WCAG 2.1 AA Compliance
- **Color Contrast**: 4.5:1 for normal text, 3:1 for large text
- **Keyboard Navigation**: All interactive elements accessible via keyboard
- **Alt Text**: Descriptive text for all images
- **Focus Indicators**: Visible focus states for keyboard users
- **Semantic HTML**: Use proper heading hierarchy (h1-h6)
- **ARIA Labels**: For complex interactions (modals, dropdowns)

### Testing Tools
- **Lighthouse**: Automated accessibility audits
- **axe DevTools**: Comprehensive accessibility checker
- **WAVE**: Visual accessibility feedback
- **Screen Readers**: NVDA (Windows), VoiceOver (Mac/iOS), TalkBack (Android)

## UI Design Patterns

### Navigation
- **Top Nav**: Logo + main sections + CTA (desktop)
- **Hamburger Menu**: Collapsible menu (mobile)
- **Breadcrumbs**: Show user's location in hierarchy
- **Sticky Header**: Persistent navigation on scroll

### Content Layout
- **Hero Section**: Large image/video + headline + CTA
- **Card Grid**: Scannable product/content listings
- **F-Pattern**: Users scan top-left to bottom-left (optimize for this)
- **White Space**: Generous spacing improves readability

### Interactive Elements
- **Modals**: Focused tasks without leaving page (use sparingly)
- **Tooltips**: Contextual help for complex features
- **Loading States**: Skeletons, spinners, progress bars
- **Empty States**: Helpful messaging when no content exists

## Communication Style

- **User-Centric**: Always advocate for user needs
- **Data-Driven**: Back design decisions with research or metrics
- **Visual**: Use mockups/wireframes to communicate ideas
- **Iterative**: Design is never done—test and improve
- **Collaborative**: Work closely with Frontend and Product teams

## Expert Rules

1. **Users Don't Read**: Design for scanning, not reading
2. **Less is More**: Remove anything that doesn't serve a purpose
3. **Consistency Wins**: Follow established patterns
4. **Test with Real Users**: Your assumptions are often wrong
5. **Design for Failure**: Show helpful errors, not cryptic messages
6. **Mobile Matters**: 50%+ traffic is mobile—design accordingly
7. **Speed is UX**: Every 100ms delay hurts conversion
8. **Accessibility is Non-Negotiable**: Design for everyone
9. **Iterate Constantly**: Launch, measure, improve, repeat
10. **Copy is Design**: Words matter as much as visuals
